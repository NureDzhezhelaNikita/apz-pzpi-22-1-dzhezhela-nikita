3 ОПИС ВИКОНАНОЇ РОБОТИ
1. Основні рівні архітектури Android:
* Applications містить користувацькі застосунки (Календар, Браузер, Viber, Пошта, Калькулятор, YouTube).
* Application Framework забезпечує інтерфейс для розробників через різні сервіси (Activity Manager, Resource Manager, Notification Manager тощо).
* Android Runtime складається з Android Runtime та бібліотек, що підтримують роботу застосунків.
* Libraries набір нативних бібліотек на C/C++ для забезпечення роботи мультимедіа, баз даних, графіки.
* Linux Kernel ядро системи, яке управляє апаратними ресурсами, процесами, пам’яттю, мережею.
2. Патерни проєктування, що використовуються:
* Singleton, для менеджерів ресурсів та служб в Application Framework.
* Observer, у Notification Manager для обробки змін стану застосунків.
* Factory, для створення об'єктів в Android Runtime.
* Proxy, у взаємодії застосунків із системними службами через Binder IPC.
3. Основні сервіси та бібліотеки:
* Activity Manager - контроль життєвого циклу застосунків.
* Location Manager - доступ до сервісів геолокації.
* View System - побудова інтерфейсів користувача.
* SQLite - база даних для збереження інформації.
* OpenGL ES - для графіки.
* Binder IPC - механізм комунікації між процесами.
